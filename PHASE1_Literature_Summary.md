## Core Observations
- Roman Urdu orthographic variation is a major source of token fragmentation.
- Code-mixed settings benefit from multilingual Transformer backbones.
- Multi-class emotion/hope tasks are significantly harder than binary tasks.
- Existing systems underuse language-aware normalization before model training.

## Gap Used in This Project
- Most baselines treat Roman Urdu as raw input.
- HLSP introduces normalization before semantic modeling to reduce embedding dilution.

## Metrics to Compare Against Literature
- Macro F1
- Weighted F1
- Per-class F1 for 4 hope classes
